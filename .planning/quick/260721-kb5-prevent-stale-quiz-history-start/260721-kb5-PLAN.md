# Prevent stale quiz start after browser history return

## Goal

Close the production history-navigation blocker discovered after PR #108: after a learner passes, leaves through the real course and dashboard links, and presses browser Back twice, the lesson must not resurrect its cached pre-attempt Start quiz screen.

## Diagnosis and constraints

- Production proof shows persisted state is correct: two completed attempts at 0 and 100, three responses each, one quiz lesson completion, and a hard reload renders the server Passed gate.
- The failure is Next App Router Client Cache reuse, not lost database state or Chrome BFCache. A true BFCache restoration would preserve the completed client result.
- Do not restore `router.refresh()` after finalization or `revalidatePath` inside finalization; both replace the required focused result card.
- Preserve the server answer-key boundary and all existing eligibility, first-answer-lock, resume, scoring, and completion behavior.
- Treat the narrow pre-existing overlapping start/finalize race as follow-up scope; the observed sequential stale start is already fail-closed.
