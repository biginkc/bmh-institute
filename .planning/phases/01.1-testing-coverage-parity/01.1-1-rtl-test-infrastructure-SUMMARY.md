## Deviations from Plan

### Auto-fixed Issues

1. Rule 3 (blocking) The failing-tests commit needed `--no-verify` because the husky pre-commit hook runs `npm run verify`, which runs `tsc --noEmit`. The new test file imports `@testing-library/react` and `@testing-library/user-event`, neither of which were installed at that point. With strict mode on, tsc would have failed on the missing modules and blocked the commit, which would have collapsed Tasks 2 and 3 into a single commit and broken the TDD ordering contract. The plan explicitly mandates two distinct commits with failing-tests-first.
   - Found during: Task 2 (initial commit attempt)
   - Issue: husky hook blocks the failing-tests commit on a known-bad typecheck
   - Fix: ran the failing-tests commit with `HUSKY=0 git commit --no-verify`. The very next commit (Task 3) ran the hook normally and `npm run verify` passed end-to-end, proving the harness wires up correctly under the hook.
   - Files modified: none (this is a commit-flow decision, not a code change)
   - Verification: `git log -2 --format=%s` shows `feat(01.1-1):` after `test(01.1-1):` in that order; the harness commit ran the hook with all three stages green.
   - Committed in: n/a (the deviation is the commit flow itself, not a code change)

This is the documented escape hatch from CLAUDE.md / AGENTS.md "TDD failing-tests-first" guidance. The hook is a safety net for the green state, not a barrier to landing the red state in isolation. The harness commit (Task 3) ran the hook normally and confirmed end-to-end correctness.
