## HIGH

### HIGH-1: Plan 3 HARDEN-01 spec asserts the wrong layer of the contract

Plan 3 Decision A drafts the HARDEN-01 spec as `test.use({ storageState: { cookies: [], origins: [] } })`, asserting that an unauthenticated GET to `/admin/reports/*` redirects to `/login`. The framing in the plan is "a clean session is the strictest test of the guard."

This is a false equivalence. The redirect chain has two layers:
- Middleware (`src/lib/supabase/middleware.ts:46-48`) redirects unauthenticated requests to `/login` before the page handler runs.
- Page-level `requireAdmin()` (`src/lib/auth/guard.ts:31-38`) redirects authenticated-but-non-admin to `/dashboard`.

The unauthenticated case exercises only the middleware. The page-level `requireAdmin()` that Phase 01-WR-05 specifically added is bypassed by an unauthenticated request because the middleware short-circuits first. If a future change deletes `await requireAdmin()` from a page body, the spec will still pass, and the regression Phase 01.1 is meant to lock will silently pass through.

The contract Phase 01 added is observable only with an authenticated learner session that reaches the page handler. CONTEXT.md decisions block line 59 says exactly this: "Sign in as a non-admin learner ... assert each navigation lands on `/dashboard`."

Fix paths (Jarrad to lock):
1. Keep the four unauthenticated cases AND add four learner-context cases that sign in as a non-admin learner. Two-layer coverage. Recommended.
2. Replace the unauthenticated cases entirely with learner-context cases. Closer to CONTEXT.md original intent. Requires provisioning a non-admin learner account on prod (one-time write).
3. Loosen TPAR-03 acceptance to "redirect away from `/admin/reports/*`" without specifying the target. Loosens the contract Phase 01 added. Not recommended.

### MEDIUM-3: Plan 3 Task 2 verify shell-pipeline bug

`01.1-3-...-PLAN.md` line 445: `grep -c "describe.skip("` will match both `describe.skip(` and `describe.skipIf(` because the dot is unescaped and `skip` is a prefix of `skipIf`. After Sub-change 2 lands, the count will be 1 (the `skipIf` line), passing the check while a stray `describe.skip` could remain.

Fix: anchor the grep with an escaped dot and an exact-prefix lookahead, e.g. `grep -cE "describe\.skip\("` and assert 0 after the change. Also fix the `! grep ... | head -5` operator-precedence ambiguity nearby.
