# Phase 01.1 — Security

Per-phase security contract: threat register, accepted risks, and audit trail.

This phase added testing infrastructure only. No production code paths or user-facing routes were modified. The threat surface is limited to test runtime, supply chain, and credential handling in test fixtures.

## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| Test runtime to repo source | RTL tests run in jsdom with file-system access; npm install pulls dep tarballs from the npm registry | Source code, npm tarball hashes |
| Test runtime to Supabase | `adminClient()` factory uses service-role key, which bypasses RLS; if pointed at the wrong project, every row is reachable | Supabase service-role JWT, row data |
| Test runtime to BMH dev server | Playwright drives a real `next dev` instance on port 3200; server-side code paths execute against whatever Supabase project the env vars point at | HTTP requests, session cookies |
| Unauthenticated client to BMH Institute prod | New HARDEN-01 spec exercises the request path of an attacker with no credentials; asserting the redirect contract IS the security check | Unauthenticated HTTP requests |

## Threat Register

| Threat ID | Category | Component | Disposition | Mitigation | Status |
|-----------|----------|-----------|-------------|------------|--------|
| T-01.1-1-01 | Tampering | New devDependencies (`@testing-library/*`, `jsdom`) | accept | Pinned exact minor versions verified against npm registry on 2026-04-30; package-lock.json pins resolved tarball hashes; same supply-chain posture as existing devDeps | closed |
| T-01.1-1-02 | Information Disclosure | RTL setup file localStorage shim | mitigate | `vitest.rtl.setup.ts` installs an in-memory `Map<string,string>` shim via `Object.defineProperty(window, "localStorage")`; `afterEach` clears the store, runs `cleanup()`, and calls `vi.clearAllMocks()`. No real browser localStorage is touched. | closed |
| T-01.1-1-03 | Tampering | RTL test discovery exclude globs | mitigate | `vitest.rtl.config.ts` `exclude: ["**/*.integration.test.ts", "node_modules/**", "e2e/**"]` prevents the RTL run from pulling in Postgres-backed tests or Playwright specs. Verified by `npm run test:rtl` running only the one smoke spec (UAT test 1, 2). | closed |
| T-01.1-2-01 | Tampering / Information Disclosure | `adminClient()` constructed against prod Supabase project | mitigate | Hard guard in `e2e/fixtures.ts` lines 27-31: throws if `TEST_SUPABASE_URL.includes("dhvfsyteqsxagokoerrx")`. Throws at construction, not on first query. Verified at runtime by UAT test 4: throw fires with the expected message. | closed |
| T-01.1-2-02 | Tampering | Storage state file `e2e/.auth/user.json` leaks credentials | accept | File contains a Supabase access token tied to the long-lived test user (`claude@test.com`) on a non-prod project. Gitignored at `.gitignore` lines 51-52. Same posture as `e2e-prod/.auth/state.json`. | closed |
| T-01.1-2-03 | Spoofing | Test user credentials in repo (`TEST_USER_PASSWORD = "test12345"`) | accept | Constants are the well-known test user that exists ONLY on a non-prod Supabase project. The prod-ref guard prevents these constants from ever being used against prod. Sandra CRM uses the same posture. | closed |
| T-01.1-2-04 | Information Disclosure | Sandra CRM helpers copy-pasted with the wrong prod ref (`copflsklaefwzipsrjqz`) | mitigate | `grep -c "copflsklaefwzipsrjqz" e2e/fixtures.ts` returns 0; `grep -c "dhvfsyteqsxagokoerrx" e2e/fixtures.ts` returns 3 (correct BMH ref). Verified inline. | closed |
| T-01.1-3-01 | Information Disclosure | `/admin/reports/*` reachable via unauthenticated session | mitigate | `e2e-prod/admin-route-guard-learner.spec.ts` asserts redirect to `/login` for each of the four routes. Verified live against bmh-institute.vercel.app by UAT test 6: 4/4 cases pass. Defence in depth: unit tests in `src/app/(dashboard)/admin/reports/**/page.test.ts` (Phase 01) assert the same contract at the function-call layer. | closed |
| T-01.1-3-02 | Information Disclosure | `01-HUMAN-UAT.md` deferral notes leak rationale | accept | Deferral cites Path A and references CONTEXT.md. No secrets, no credentials, no internal URLs. File remains in `.planning/` which is committed. | closed |
| T-01.1-3-03 | Tampering | `docs/test-environment-setup.md` instructs contributors how to obtain service-role keys | mitigate | Runbook describes the Dashboard path only; key-value scan (regex match for JWTs, Stripe-like keys, long hex tokens) returns no matches. Explicit warning "Treat this like a password" present. The fixtures' prod-ref guard (T-01.1-2-01) is the runtime enforcement. | closed |
| T-01.1-3-04 | Denial of Service | `describe.skipIf(!envPresent)` causes silent regressions to be reported as skipped | accept | Previous behaviour was strictly worse: `describe.skip` silently never ran; import-time throw killed the entire integration run. `describe.skipIf` reports `skipped` cleanly (verified by UAT test 5). CI does not currently run integration; the contributor running `npm run test:integration` is expected to verify env is populated before relying on the result. | closed |

Status: open or closed
